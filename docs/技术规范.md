# 「XP复古人生四格照相馆」技术规范

> 版本：v1.0 | 日期：2026-07-13

---

## 一、技术选型

### 为什么纯前端？

| 考量 | 理由 |
|------|------|
| 用户是小白的 | 无法配置服务器/数据库 |
| 隐私优先 | 用户照片不上传任何服务器 |
| Windows 环境 | 只需浏览器，不装任何软件 |
| 维护成本 | 一个 HTML 文件，零运维 |

### 为什么不用框架？

React/Vue 需要构建工具链（Node.js/npm），对小白不友好。单文件原生 HTML+CSS+JS 双击就能跑。

## 二、关键技术 API

### 1. 摄像头访问 —— `getUserMedia`

```javascript
// 调用电脑摄像头
const stream = await navigator.mediaDevices.getUserMedia({
  video: { width: 1280, height: 720, facingMode: 'user' }
});
videoElement.srcObject = stream;
```

- **要求**：HTTPS 或 localhost（本地 HTML 文件直接打开时浏览器允许）
- **兼容性**：Chrome 53+ / Edge 79+

### 2. Canvas 图像合成 —— `CanvasRenderingContext2D`

```javascript
const canvas = document.createElement('canvas');
const ctx = canvas.getContext('2d');

// 绘制图片
ctx.drawImage(image, x, y, width, height);

// 圆角裁剪
ctx.beginPath();
ctx.roundRect(x, y, w, h, radius);
ctx.clip();

// 导出
canvas.toBlob(blob => { /* 下载 */ }, 'image/png');
```

### 3. 滤镜 —— `getImageData / putImageData`

```javascript
const imageData = ctx.getImageData(0, 0, w, h);
const pixels = imageData.data; // Uint8ClampedArray [R,G,B,A, R,G,B,A, ...]

for (let i = 0; i < pixels.length; i += 4) {
  // 对每个像素的 R/G/B 做变换
  pixels[i]     = transformR(pixels[i]);     // Red
  pixels[i + 1] = transformG(pixels[i + 1]); // Green
  pixels[i + 2] = transformB(pixels[i + 2]); // Blue
  // pixels[i + 3] = Alpha
}
ctx.putImageData(imageData, 0, 0);
```

### 4. 文件读取 —— `FileReader`

```javascript
const file = event.target.files[0];
const reader = new FileReader();
reader.onload = (e) => {
  const img = new Image();
  img.src = e.target.result;
};
reader.readAsDataURL(file);
```

### 5. 文件下载 —— `URL.createObjectURL`

```javascript
canvas.toBlob(blob => {
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = '人生四格_' + timestamp + '.png';
  a.click();
  URL.revokeObjectURL(url);
});
```

### 6. 本地存储 —— `IndexedDB`（Phase 3 使用）

Phase 1 暂不使用。图片导出直接用下载功能。

## 四、Canvas 合成流程

```
素材1 → [缩放/裁剪] → Canvas1
素材2 → [缩放/裁剪] → Canvas2
素材3 → [缩放/裁剪] → Canvas3
素材4 → [缩放/裁剪] → Canvas4
                            ↓
              主Canvas 按模板参数合成
              （间距、圆角、边框、背景色）
                            ↓
                    应用滤镜处理
                            ↓
                    最终输出 Canvas
                            ↓
                    导出 / 预览显示
```

## 五、性能优化策略

| 策略 | 说明 |
|------|------|
| Canvas 离屏渲染 | 所有图像处理在不可见 Canvas 上完成，最后一次性绘制到屏幕 |
| 滤镜防抖 | 滑块拖拽时 100ms 防抖，避免频繁重绘 |
| 缩略图预处理 | 上传原图后先生成 200×200 缩略图用于 UI 展示 |
| 懒加载 | 滤镜着色表预计算，避免重复创建数组 |
